# 🧠 核心概念

> 用大白话解释燃灯是怎么工作的。

---

## 整体工作流

```
你(人) ──大纲/人设──▶ 青灯(策划编辑) ──写作指令──▶ 落笔(写手) ──正文──▶ 你(人)
                                                                      │
                                                                      ▼
                                                                  审稿(校对)
```

---

## 青灯 — 你的策划编辑

**青灯**（即 `openwrite goethe`）是你的 AI 策划编辑。

它做什么：

- 和你讨论剧情走向、人物弧光
- 帮你把灵光一闪扩展成可写大纲
- 确保故事线不出现漏洞
- 主动建议伏笔和高潮布局

怎么用：

```bash
openwrite goethe
# 进入对话模式，像跟编辑聊天一样：
# "我想写一个被退婚的少年遇到神秘玉佩的故事..."
```

**记住：** 青灯只动脑，不动笔。它产出的是策划方案，不是正文。

---

## 落笔 — 你的专属枪手

**落笔**（即 `openwrite write`）是你的 AI 写手。

它做什么：

- 严格按你的大纲写正文
- 始终维护人物一致性（不会把主角名字写错）
- 自动延续你的写作风格（用「风格指纹」技术）
- 一章一章地写，按你的节奏来

怎么用：

```bash
openwrite write next       # 写下一章
openwrite write ch_003     # 写指定章节
```

**和直接让 ChatGPT 写的区别：** 落笔认识你的整本书。它记得第 3 章埋的伏笔，不会在第 20 章忘掉。它知道主角在第 5 章已经突破筑基期了，不会让他重生回去。

---

## 人设（风格指纹）— 你的创作指纹

每本书都有自己的「味道」。燃灯通过三层机制确保所有章节读起来像同一个人写的：

### 1. 风格指纹 (Style Fingerprint)

自动从你的样章中提取：

- **语感节奏** — 句子长短、段落分布
- **对话习惯** — 人物怎么说话（"……" 多不多，有没有口头禅）
- **描写偏好** — 喜欢写环境还是写心理，比喻风格
- **词汇偏好** — "说" 和 "道" 的比例，古风浓度

### 2. 角色一致性

每个角色都有独立的档案，记录：

- 性格标签和说话方式
- 与其他角色的关系
- 经历的重要事件（不会随时间遗忘）

### 3. 真相档案 (Truth Files)

燃灯会在写作过程中自动维护一系列「真相文件」：

| 文件 | 记录什么 |
|------|---------|
| `current_state` | 当前剧情进度、世界观状态 |
| `ledger` | 时间线、各角色位置和状态 |
| `relationships` | 角色关系网络和变化 |

---

## 审稿 — 自动校对

写完之后，燃灯会帮你审查章节质量。

检查什么：

| 检查项 | 说明 |
|--------|------|
| 🚩 AI 痕迹 | 避免"值得注意的是""综上所述"等 AI 套话 |
| 🎭 人物 OOC | 角色行为是否符合其性格设定 |
| 🔗 逻辑漏洞 | 前后情节是否自洽 |
| 📖 节奏问题 | 是否该推进剧情的时候在灌水 |
| 🔮 伏笔管理 | 提醒你之前埋的伏笔该回收了 |

怎么用：

```bash
openwrite review latest     # 审查最新章节
openwrite review ch_003     # 审查指定章节
```

---

## 工作流 — 从大纲到完本

燃灯的完整创作流程：

```
构思阶段         规划阶段         写作循环         收尾阶段
┌────────┐     ┌────────┐     ┌─────────┐     ┌────────┐
│ 灵感   │ ──▶ │ 大纲   │ ──▶ │ 写作    │ ──▶ │ 导出   │
│ 设定   │     │ 人设   │     │ 审稿    │     │ 发布   │
│ 市场   │     │ 世界   │     │ 修改    │     │        │
└────────┘     └────────┘     └──┬──────┘     └────────┘
                                 │ 循环直到
                                 │ 全部写完
                                 ▼
```

## 常用命令一览

| 命令 | 谁 | 干什么 |
|------|-----|--------|
| `openwrite setup` | — | 配置 AI 模型 |
| `openwrite init` | — | 创建新小说项目 |
| `openwrite goethe` | 青灯 | 和策划编辑聊大纲和角色 |
| `openwrite write next` | 落笔 | 写下一章 |
| `openwrite review latest` | 审稿 | 审查最新章节 |
| `openwrite desk` | — | 打开小说工作台 |
| `openwrite sync` | — | 同步 src 到 data |
| `openwrite export` | — | 导出整本书 |
| `openwrite status` | — | 查看项目状态 |
